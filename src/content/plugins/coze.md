# Coze 扣子空间批量删除项目列表

如果你用过 Coze 搭智能体，大概率遇到过这个问题：项目列表里堆积了几十个测试项目，想清理却发现 Coze 只支持**逐个删除**——点三个点、选删除、输项目名确认、等页面刷新，重复 N 次。

## 它能做什么

一句话：**自动批量删除 Coze 平台上的项目/智能体。**

具体流程：

1. 打开 Coze 个人空间的项目列表页
2. 点击右下角悬浮球，弹出操作面板
3. 点"扫描项目"，插件会自动滚屏加载所有卡片，提取项目名称
4. 勾选要删的项目（支持全选）
5. 点"开始删除"，剩下的事情全部自动完成
![image.png](https://ymkit-1307887579.cos.ap-chengdu.myqcloud.com/img/20260615125838500.webp)


删除过程的每一步——悬停卡片、点击三个点按钮、在下拉菜单中找到"删除"选项、等待确认弹窗、填写项目名称、点击确认按钮——全部由插件模拟操作，不需要人工介入。

进度实时显示，随时可以点"停止执行"中断。删完后给出汇总：成功几个、失败几个，失败的原因是什么。

## 安装和使用

### 安装

1. 下载插件压缩包并解压
2. 打开 Chrome，地址栏输入 `chrome://extensions/`
3. 打开右上角"开发者模式"
4. 点击"加载已解压的扩展程序"，选择解压后的文件夹或者直接拖进来

### 使用

1. 访问 `https://www.coze.cn/space/*/develop`（你的 Coze 项目列表页）
2. 页面右侧会出现一个蓝色悬浮球，点它
3. 点"扫描项目"
4. 勾选要删除的项目
5. 点"开始删除"


## 技术实现上的几个有意思的点

如果你对浏览器插件开发感兴趣，下面几个实现细节值得一看。

### 1. DOM 扫描的三层策略

Coze 的页面结构可能随版本变化，所以项目卡片的查找用了三层策略：

- **策略一**：按已知的 `data-testid` 或 CSS class 精确匹配
- **策略二**：查找 grid 布局容器的直接子元素
- **策略三**：全量扫描页面上所有 DOM 节点，按尺寸、是否包含按钮/链接来筛选

三层逐级降级，确保不同版本的页面都能扫到卡片。

### 2. Semi Design Portal 的坑

Coze 用的是 Semi Design 组件库，它的下拉菜单和弹窗中的输入框是通过 Portal 渲染到 `<body>` 下的，不在触发按钮的 DOM 子树内。

这意味着不能简单地 `card.querySelector()` 来找菜单项——得全局搜索 `[role="menuitem"]` 或 `.semi-dropdown-item`，然后检查可见性。

弹窗中的确认输入框同理：先搜 `modal` 内部，搜不到再全局搜所有可见 dialog 内的 input，最后兜底取当前焦点元素。

### 3. 删除后 DOM 引用失效

每删一个项目，Coze 页面会刷新列表，之前保存的 DOM 元素引用全部失效。

解决方式是每删除一个项目后重新扫描页面，用项目名称匹配刷新剩余项目的 `element` 引用。同时在 `deleteSingleProject` 里加了 `isConnected` 兜底检测——如果发现元素已脱离 DOM，按名称重新查找对应卡片。

### 4. disabled 按钮的处理

删除确认按钮在校验通过前是 `disabled` 状态。插件填完项目名后，如果按钮还没解禁，直接 `removeAttribute('disabled')` 强制激活，不依赖页面的响应式校验时机。
